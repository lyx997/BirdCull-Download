# 第三方组件声明

BirdCull 以闭源商业软件/专有软件的方式发布。详见 [许可声明](LICENSE.md)。

当前 Windows 发布包同时包含或依赖若干第三方开源组件。这些组件仍分别受其各自许可证、版权声明和署名要求约束。

基于对当前仓库和打包流程中核心依赖的核查，BirdCull 目前看起来适合以闭源二进制形式发布。就当前已核查的核心打包依赖而言，没有发现某个核心依赖会仅仅因为你发布 BirdCull 的二进制安装包，就当然要求公开 BirdCull 自身源码。

这是一份面向发布操作的实际说明，不构成正式法律意见。如果你后续要做商业分销、转售、OEM 捆绑或企业采购合规审查，建议再做一次专业法律审核。

## 当前已核查的核心组件

- `RF-DETR` (`rfdetr`) - Apache License 2.0
- `PyTorch` (`torch`, `torchvision`) - BSD 风格许可证
- `NumPy` - BSD 风格许可证
- `OpenCV` (`opencv-python`) - Apache License 2.0
- `cryptography` - 宽松开源许可证体系
- `PyInstaller` - GPL，但附带 PyInstaller 例外条款，可用于构建和分发闭源应用

## 这对 BirdCull 发布意味着什么

- BirdCull 自身并未在本仓库中作为开源软件发布。
- 这个发布仓库主要用于提供安装包、版本说明和面向最终用户的信息。
- 第三方开源组件仍保留其各自的许可证约束。
- 如果相关许可证要求保留声明或署名信息，发布材料中应继续保留这些内容。

如果未来版本新增依赖、替换运行时库，或更改分发方式，这份声明也应重新核查一次。
